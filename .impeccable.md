## Design Context

### Users
End users supervising an AI assistant in production products. Glass Box is intended to become an open source framework that product teams can add to shipped AI experiences as an explainability layer.

### Brand Personality
Calm, utilitarian, and precise. The UI should help people understand and control AI reasoning without feeling like a developer debugger or a decorative visualization.

### Aesthetic Direction
Neutral product UI inspired by Codex's Progress module: matte dark surfaces, subtle borders, restrained depth, and sparse accent color only where it clarifies active state or risk. The spatial rail should feel like a practical reasoning workspace, not a graph toy.

### Design Principles
- Keep the active reasoning chain centered, prominent, and legible.
- Preserve branch positions in a stable spatial canvas; switching branches pans focus instead of reordering history.
- Push inactive branches into the background as ghosted summaries until the user chooses them.
- Use progressive disclosure for explanatory copy and secondary details.
- Favor sober, high-contrast utility over glow, decoration, or excessive color.
